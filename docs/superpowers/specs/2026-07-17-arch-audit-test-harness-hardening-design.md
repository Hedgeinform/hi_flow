# Дизайн: исправление долгов тестовой обвязки ArchAudit

**Дата:** 2026-07-17
**Статус:** утверждён оператором, ожидает реализации
**Контур:** `hi_flow/skills/arch-audit/`

## Контекст

После исправления JavaScript/ESM-регрессии полный прогон выявил четыре старых дефекта тестовой обвязки:

1. часть тестов разрешает `tests/fixtures/...` через текущий рабочий каталог;
2. четыре интеграционных теста пишут отчёты в версионируемые каталоги фикстур;
3. D9-парсер и Markdown-фикстуры чувствительны к CRLF в Windows worktree;
4. AJV игнорирует `format: date-time`, поэтому `audit_timestamp` фактически не валидируется.

Первый и второй дефекты нарушают изоляцию тестов. Третий делает результат зависимым от платформы. Четвёртый ослабляет уже объявленный D8-контракт. Нового пользовательского поведения эта работа не вводит.

## Принятое решение

Исправить причины в каждом слое, не использовать глобальную смену `process.cwd()` и не создавать новый универсальный тестовый фреймворк.

### Независимые пути к фикстурам

Создать маленький тестовый helper, который вычисляет корень пакета и пути к фикстурам относительно собственного `import.meta.url`. Тесты, которые сейчас передают строковые пути `tests/fixtures/...` или строят их от `process.cwd()`, переводятся на helper.

`vitest.config.ts` получает явный корень пакета, чтобы discovery тестов и конфигурация также не зависели от места запуска. Боевой код не импортирует тестовый helper.

### Изолированные интеграционные артефакты

Интеграционные тесты используют исходные проекты в `tests/fixtures/<project>` только для чтения. Каждый тест создаёт отдельный выходной каталог через `mkdtemp()` в системной временной директории, передаёт его как `outDir` и удаляет в `finally`.

Тест дополнительно фиксирует, что tracked-каталог `audit-report/` соответствующей фикстуры не изменился. Обновление эталонных снимков, если оно когда-либо потребуется, остаётся отдельной явной операцией и не выполняется обычным тестовым прогоном.

### Переводы строк D9

`loadD9()` нормализует входной Markdown `\r\n` и одиночные `\r` в `\n` сразу после чтения. Это устраняет платформенную зависимость на границе парсера.

В корневой `.gitattributes` фиксируется `eol=lf` для Markdown-фикстур ArchAudit. Нормализация в коде остаётся необходимой защитой для внешних D9-документов, которые могут приходить с CRLF независимо от Git.

### Полная D8 date-time валидация

Добавить runtime-зависимость `ajv-formats` и зарегистрировать её на существующем экземпляре AJV до компиляции D8-схемы. Сама схема и её версия не меняются: `format: date-time` уже является частью контракта.

Невалидный `audit_timestamp` должен отклоняться с ошибкой формата. Валидные RFC 3339 значения, которые уже генерирует `new Date().toISOString()`, продолжают проходить.

## Проверки

Реализация выполняется через четыре независимых RED → GREEN цикла:

1. тестовый запуск из каталога вне `arch-audit` сначала не находит фикстуры, затем проходит;
2. интеграционный прогон сначала изменяет tracked-отчёты, затем оставляет их неизменными;
3. D9-тест с CRLF сначала теряет альтернативы исправления, затем даёт тот же результат, что LF;
4. D8-тест с невалидной датой сначала ошибочно проходит, затем отклоняется.

Финальные ворота:

- целевые тесты четырёх исправлений;
- полный `vitest run` из каталога пакета;
- полный запуск через явный `vitest.config.ts` из внешнего рабочего каталога;
- `tsc --noEmit`;
- `git diff --check`;
- проверка чистоты tracked-фикстур после тестов;
- независимое ревью без незакрытых Critical/Important.

## Изменяемые артефакты

- `hi_flow/skills/arch-audit/vitest.config.ts`;
- новый тестовый helper под `hi_flow/skills/arch-audit/tests/`;
- тесты, использующие относительные fixture paths;
- четыре существующих integration-теста и JS/ESM integration-тест для единого шаблона cleanup;
- `hi_flow/skills/arch-audit/core/d9-loader.ts` и его тест;
- `hi_flow/skills/arch-audit/core/d8-schema-validator.ts` и его тест;
- `hi_flow/skills/arch-audit/package.json` и `package-lock.json`;
- корневая `.gitattributes`;
- `docs/active-issues.md`, откуда после полной проверки удаляется закрытый LOW-долг о мутации фикстур;
- `PROJECT_STATE.md` и отчёт реализации рядом с implementation plan.

## Не входит в работу

- изменение D8 schema version или структуры отчёта;
- автоматическая команда обновления snapshot-файлов;
- реорганизация всего тестового дерева;
- устранение других Active Issues ArchAudit;
- добавление lint/CI baseline.

## Риски и ограничения

- `ajv-formats` — новая небольшая runtime-зависимость; она устанавливается стандартным `npm install` с обновлением lock-файла.
- Явный корень Vitest не заменяет абсолютные fixture paths: оба слоя нужны, потому что discovery и файловый ввод — разные границы.
- Очистка временных каталогов выполняется только для директорий, созданных самим тестом.

## Критерии готовности

- запуск тестов не зависит от текущего каталога;
- обычный integration-прогон не меняет ни один tracked-файл;
- LF и CRLF D9 дают идентичный индекс;
- невалидный `audit_timestamp` отклоняется без AJV warning;
- полный пакет зелёный, рабочее дерево после него содержит только намеренные изменения.
